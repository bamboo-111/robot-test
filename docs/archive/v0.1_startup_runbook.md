# v0.1-alpha 启动与运行手册

## 1. 适用范围

本文面向参与 Kuavo5-W / MuJoCo / ROS / Docker / WSL2 / Web 控制台协作的 A/B/C 用户。它用于从 Windows 主机重启后的状态，逐步恢复到可以打开 Web 控制台并执行安全探测的状态。

角色建议：

- A：主控和环境维护者，可以启动 WSL、Docker、容器和仿真。
- B/C：协作者，优先通过 SSH 和 Web 控制台观察、导入安全脚本、运行指定 Demo，并记录问题。

禁止事项：

- B/C 不要重启 Windows、WSL、Docker 服务。
- B/C 不要删除 Docker 容器、镜像、volume。
- B/C 不要直接发布 ROS 控制命令。
- B/C 不要运行未知来源脚本。
- B/C 不要长时间发布非零 `/cmd_vel`。

## 2. 系统结构

```text
Windows 主机
  -> WSL2 Ubuntu-20.04
  -> Docker 容器 kuavo5w_sim
  -> ROS / MuJoCo / Kuavo SDK
  -> Web 控制台 127.0.0.1:8765
```

已知路径：

```text
Windows 项目目录：E:\project\kuavo
WSL 项目目录：/mnt/e/project/kuavo
容器内项目挂载：/root/kuavo_deploy
容器内 Kuavo 工作区：/root/kuavo_ws
```

## 3. 启动前检查

### 3.1 Windows 侧检查

运行位置：Windows PowerShell。

目的：确认项目目录存在，Windows 可以调用 WSL。

命令：

```powershell
cd E:\project\kuavo
wsl -l -v
```

预期结果：

- 能进入 `E:\project\kuavo`。
- `wsl -l -v` 能看到 `Ubuntu-20.04`。

失败时查看：

- `docs/v0.1_recovery_checklist.md` 中的 “WSL 进入失败”。
- 如果 WSL 不存在或无法启动，联系 A。

### 3.2 WSL 侧检查

运行位置：Windows PowerShell。

目的：进入 WSL 并确认项目挂载路径。

命令：

```powershell
wsl -d Ubuntu-20.04
```

进入 WSL 后运行：

```bash
cd /mnt/e/project/kuavo
pwd
docker ps -a
```

预期结果：

- `pwd` 输出 `/mnt/e/project/kuavo`。
- `docker ps -a` 能看到容器列表。
- 正常情况下存在 `kuavo5w_sim`。

失败时查看：

- `docs/v0.1_recovery_checklist.md` 中的 “Docker 容器 kuavo5w_sim 不存在或未运行”。
- Docker CLI 不可用时联系 A。

### 3.3 Docker 容器检查

运行位置：WSL Ubuntu。

目的：确认容器存在并处于 running 状态。

命令：

```bash
docker ps -a --filter name=kuavo5w_sim
```

预期结果：

- 输出中包含 `kuavo5w_sim`。
- `STATUS` 包含 `Up`。

如果容器不存在或未运行：

- B/C 不要自行删除或重建容器。
- A 可按项目脚本检查启动流程：

```bash
cd /mnt/e/project/kuavo
bash scripts/wsl/run-kuavo-container-windows-gui.sh
```

失败时查看：

- `docs/v0.1_recovery_checklist.md`。
- `docs/windows-kuavo5w-docker.md`。

### 3.4 启动平台和仿真

运行位置：Windows PowerShell。

目的：启动或复用 `kuavo5w_sim` 容器，并在容器内启动 MuJoCo / ROS。仅看到 Docker 容器 `STATUS` 为 `Up` 不代表 ROS master、MuJoCo 或控制节点已经启动。

推荐由 A 执行：

```powershell
cd E:\project\kuavo
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\windows\start-kuavo5w-platform.ps1
```

如果只需要执行安全探测：

```powershell
cd E:\project\kuavo
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\windows\start-kuavo5w-platform.ps1 -RunProbe
```

预期结果：

- PowerShell 输出 `Starting MuJoCo launch in container background...` 或 `MuJoCo launch already appears to be running.`。
- PowerShell 输出 `Waiting for ROS interfaces...`。
- 成功时输出 `ROS interfaces are ready.`。

如果等待 ROS interfaces 超时：

```powershell
wsl -d Ubuntu-20.04 -- docker exec -it kuavo5w_sim bash -lc 'tail -120 /tmp/kuavo5w_mujoco_start.log'
```

失败时查看：

- `docs/v0.1_recovery_checklist.md` 中的 “rostopic list 失败”、“仿真无响应” 和 “GUI 不显示”。
- `docs/windows-kuavo5w-docker.md` 中的平台启动说明。

### 3.5 ROS topic 检查

运行位置：WSL Ubuntu。

目的：在平台和仿真启动后，确认容器内 ROS 命令可用，并检查关键 topic/service 是否存在。

注意：

- 只有 Docker 容器处于 `Up` 状态时，ROS master 可能仍未启动。
- 如果这里出现 `ERROR: Unable to communicate with master!`，通常表示还没有完成第 3.4 节的平台和仿真启动，或启动失败。

命令：

```bash
docker exec kuavo5w_sim bash -lc "source /root/kuavo_ws/installed/setup.bash; source /root/kuavo_ws/devel/setup.bash; rostopic list | sort | head -80"
docker exec kuavo5w_sim bash -lc "source /root/kuavo_ws/installed/setup.bash; source /root/kuavo_ws/devel/setup.bash; rostopic info /cmd_vel"
docker exec kuavo5w_sim bash -lc "source /root/kuavo_ws/installed/setup.bash; source /root/kuavo_ws/devel/setup.bash; rosservice info /mobile_manipulator_mpc_control"
```

预期结果：

- `rostopic list` 能输出 topic 列表。
- `/cmd_vel` 能显示 topic 信息。
- `/mobile_manipulator_mpc_control` 能显示 service 信息。

失败时查看：

- `docs/v0.1_recovery_checklist.md` 中的 `/cmd_vel`、`/mobile_manipulator_mpc_control`、`rostopic list` 相关项。

## 4. 启动 Web 控制台

运行位置：Windows PowerShell。

目的：启动本地 Web 控制台服务。

命令：

```powershell
cd E:\project\kuavo
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\windows\start-web.ps1 -Port 8765
```

预期结果：

- PowerShell 窗口中 Web 控制台保持运行。
- 没有立即退出或报错。

失败时查看：

- `docs/v0.1_recovery_checklist.md` 中的 “Web 控制台打不开” 和 “127.0.0.1:8765 无响应”。

## 5. 打开 Web 控制台

运行位置：浏览器 Web 控制台。

目的：打开控制页面。

地址：

```text
http://127.0.0.1:8765/
```

预期结果：

- 页面显示 Kuavo Web 控制台。
- 能看到启动、连接、脚本导入、运行脚本、日志等区域。

失败时查看：

- 确认 `scripts\windows\start-web.ps1` 的 PowerShell 窗口仍在运行。
- 查看 `docs/v0.1_recovery_checklist.md`。

## 6. 连接或启动仿真

运行位置：浏览器 Web 控制台。

目的：让 Web 控制台连接到第 3.4 节已经启动的仿真；如果仿真尚未启动，由 A 先按第 3.4 节启动。

操作：

1. B/C 先确认没有其他人正在控制。
2. 优先点击 Web 控制台里的连接或低负载连接入口。
3. 如果页面提示 ROS 未 ready、`rostopic list` 失败或无法连接 ROS master，回到第 3.4 节检查平台启动日志。

预期结果：

- 控制台日志显示容器、ROS 或仿真接口可用。
- `/cmd_vel` 和 `/mobile_manipulator_mpc_control` 可检查通过。

失败时查看：

- `docs/v0.1_recovery_checklist.md`。
- Web 控制台日志区域。

## 7. 执行安全探测

运行位置：浏览器 Web 控制台。

目的：只读检查 ROS 状态，不移动机器人。

操作：

1. 在脚本导入区域上传 `scripts/import_examples/bc_safe_probe.py`。
2. 运行该脚本。

预期结果：

```text
[INFO] BC safe probe started
[PASS] rostopic list succeeded
[PASS] /cmd_vel found
[PASS] /mobile_manipulator_mpc_control found
[INFO] Probe completed successfully
```

失败时查看：

- 复制 Web 控制台输出。
- 填写 `docs/v0.1_bc_feedback.template.md`。

## 8. 执行前进 Demo

运行位置：浏览器 Web 控制台。

目的：在 A 确认安全后执行项目内置前进 Demo。

操作：

1. 确认仿真窗口可见。
2. 确认没有其他人正在控制。
3. 在 Web 控制台选择前进 Demo 或对应 YAML 场景。

预期结果：

- 机器人执行短时间低速前进。
- 结束后底盘停止。

失败时查看：

- `docs/v0.1_recovery_checklist.md` 中的 “前进 Demo 没有效果”。

## 9. 暂停与恢复 MPC

运行位置：浏览器 Web 控制台。

目的：通过 Web 控制台执行授权的 MPC pause/resume。

操作：

1. 点击暂停 MPC。
2. 观察日志输出。
3. 点击恢复 MPC。
4. 观察日志输出。

预期结果：

- 控制台返回成功信息。
- 仿真状态没有异常。

失败时查看：

- `docs/v0.1_recovery_checklist.md` 中的 “MPC 暂停/恢复失败”。

## 10. 导入并运行安全脚本

运行位置：浏览器 Web 控制台。

目的：导入受控、安全、可审查的 Python 脚本。

要求：

- 文件名必须是安全 ASCII `.py`。
- 单个脚本不超过 512 KiB。
- 不上传未知来源脚本。
- 不上传会发布长时间非零 `/cmd_vel` 的脚本。

推荐脚本：

```text
scripts/import_examples/bc_safe_probe.py
```

失败时查看：

- `docs/kuavo-web-import-script-guide.md`。
- `docs/v0.1_bc_feedback.template.md`。

## 11. 查看日志

运行位置：浏览器 Web 控制台。

目的：查看最近命令输出和故障信息。

操作：

1. 在 Web 控制台查看日志区域。
2. 复制失败命令输出。
3. 截图或记录时间点。

预期结果：

- 能看到脚本运行输出、错误信息或超时信息。

失败时查看：

- `docs/v0.1_recovery_checklist.md` 中的 “日志找不到”。

## 12. 安全停止与复原

运行位置：浏览器 Web 控制台或 Windows PowerShell。

目的：在授权范围内停止动作或恢复到安全状态。

Web 控制台操作：

1. 点击停止或复原入口。
2. 等待日志返回。

Windows PowerShell 命令：

```powershell
cd E:\project\kuavo
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\windows\kuavo5w-restore.ps1
```

预期结果：

- 底盘停止。
- 控制台返回复原或停止结果。

失败时查看：

- `docs/v0.1_recovery_checklist.md`。
- 必要时联系 A。

## 13. 常见注意事项

- Web 控制台是 B/C 的推荐操作入口。
- SSH 用于观察和记录，不用于绕过 Web 控制台直接控制 ROS。
- 每次控制前确认没有其他协作者正在控制。
- 任何失败都先保留输出，再查 recovery checklist。
- 反馈请使用 `docs/v0.1_bc_feedback.template.md`。
- A/B/C 验收结果由实际操作者填写，不由自动工具代填。
