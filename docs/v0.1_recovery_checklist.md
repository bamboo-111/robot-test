# v0.1-alpha 故障恢复 Checklist

本文面向 B/C 协作者。所有恢复动作默认以只读检查和 Web 控制台操作为主。遇到需要重启 WSL、Docker、容器、仿真或修改系统配置的情况，必须找 A。

禁止 B/C 执行：

- 删除 Docker 容器、镜像、volume。
- 重启 Windows、WSL、Docker Desktop。
- 直接 kill ROS、MuJoCo、Docker、控制节点进程。
- 直接发布 ROS 控制命令。
- 运行未知来源脚本。
- 长时间发布非零 `/cmd_vel`。

| 问题 | 检查命令 | 可能原因 | 恢复动作 | 是否需要 A |
|---|---|---|---|---|
| Web 控制台打不开 | Windows PowerShell: `Test-NetConnection 127.0.0.1 -Port 8765` | Web 服务未启动；端口被占用；浏览器地址错误 | 确认地址为 `http://127.0.0.1:8765/`；检查启动 Web 的 PowerShell 窗口是否报错 | 如果服务未启动或端口冲突，需要 A |
| `127.0.0.1:8765` 无响应 | Windows PowerShell: `cd E:\project\kuavo; powershell -NoProfile -ExecutionPolicy Bypass -File scripts\windows\start-kuavo5w-web-control.ps1 -Port 8765` | Web 控制台没有运行；Python 环境异常 | B/C 记录报错；不要自行改脚本 | 是 |
| SSH 无法连接 | Windows PowerShell: `ssh -v -i "$env:USERPROFILE\.ssh\<自己的私钥文件>" -p 3222 bamboo@192.168.1.109` | 不在同一局域网；A 跳板未开；公钥未加入；端口不通 | 确认 Wi-Fi 和 IP；把错误输出发给 A | 是 |
| WSL 进入失败 | Windows PowerShell: `wsl -l -v` | WSL 未启动；Ubuntu-20.04 状态异常 | B/C 不重启 WSL；记录输出 | 是 |
| Docker 容器 `kuavo5w_sim` 不存在或未运行 | WSL: `docker ps -a --filter name=kuavo5w_sim` | 容器未创建；Docker 未运行；容器退出 | B/C 不删除或重建容器；把 `docker ps -a` 输出发给 A | 是 |
| `rostopic list` 失败 | WSL: `docker exec kuavo5w_sim bash -lc "source /root/kuavo_ws/installed/setup.bash; source /root/kuavo_ws/devel/setup.bash; rostopic list"` | ROS master 未启动；环境未 source；仿真未运行 | 在 Web 控制台尝试连接或查看日志；记录错误 | 通常需要 A |
| `/cmd_vel` 不存在 | WSL: `docker exec kuavo5w_sim bash -lc "source /root/kuavo_ws/installed/setup.bash; source /root/kuavo_ws/devel/setup.bash; rostopic info /cmd_vel"` | 底盘控制节点未启动；仿真未完成初始化 | 先运行安全探测；不要直接发布速度命令 | 是 |
| `/mobile_manipulator_mpc_control` 不存在 | WSL: `docker exec kuavo5w_sim bash -lc "source /root/kuavo_ws/installed/setup.bash; source /root/kuavo_ws/devel/setup.bash; rosservice info /mobile_manipulator_mpc_control"` | MPC 服务未启动；仿真未完成初始化 | 记录输出；不要反复点击 pause/resume | 是 |
| 脚本上传失败 | 浏览器 Web 控制台；检查脚本文件名和大小 | 文件名不是安全 ASCII；不是 `.py`；超过 512 KiB | 改用 `scripts/import_examples/bc_safe_probe.py`；保留错误提示 | 否，反复失败时找 A |
| 脚本运行超时 | 浏览器 Web 控制台日志 | 脚本阻塞；ROS 无响应；仿真睡眠 | 不重复运行；保存输出；填写反馈模板 | 是 |
| 仿真无响应 | Web 控制台日志；WSL: `docker ps -a --filter name=kuavo5w_sim` | MuJoCo/ROS 卡住；容器负载高；GUI 不可见 | B/C 不 kill 进程；记录时间和日志 | 是 |
| GUI 不显示 | WSL: `docker exec kuavo5w_sim bash -lc "glxinfo -B 2>/dev/null | head -40"` | Windows X/WSLg/GPU 渲染异常；容器 DISPLAY 配置异常 | 记录输出；不要重建容器 | 是 |
| 前进 Demo 没有效果 | Web 控制台日志；ROS topic 检查 `/cmd_vel` | ROS 未 ready；模式未切换；仿真未连接 | 先运行 `bc_safe_probe.py`；不要加大速度或延长时间 | 是 |
| MPC 暂停/恢复失败 | Web 控制台日志；检查 `/mobile_manipulator_mpc_control` | 服务不存在；服务调用失败；仿真状态异常 | 停止重复点击；记录日志 | 是 |
| 日志找不到 | 浏览器 Web 控制台日志区域；Windows PowerShell 启动窗口 | Web 页面刷新；服务输出在 PowerShell 窗口；脚本没有输出 | 截图当前页面；复制 PowerShell 输出 | 否，缺关键日志时找 A |

## 反馈记录

遇到无法继续的问题时，复制命令输出或截图，并填写：

```text
docs/v0.1_bc_feedback.template.md
```

不要把真实验收结果填入模板文件本身；请复制模板另存为实际记录。
